# Contributing to Quran Desktop

First off, thank you for considering contributing to Quran Desktop! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## How to Contribute

### Reporting Bugs
If you find a bug in the application, please open an issue in our GitHub repository using the Bug Report template. Include as much detail as possible:
* Your operating system and version.
* The version of Quran Desktop you are using.
* Clear steps to reproduce the bug.
* Any relevant screenshots or error logs.

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue using the Feature Request template and describe:
* What the feature is and why it's needed.
* How it should work from a user's perspective.
* Any UI mockups or inspiration if applicable.

### Pull Requests
We welcome Pull Requests (PRs) for bug fixes, new features, and documentation improvements.
1. Fork the repository and create your branch from `main`.
2. Ensure you have followed the setup instructions in the `README.md`.
3. If you've added code that should be tested, add tests.
4. If you've changed APIs, update the documentation.
5. Ensure the test suite passes.
6. Make sure your code lints.
7. Issue that PR!

## Development Guidelines
* Use standard React functional components and hooks.
* Maintain the aesthetic quality of the application using TailwindCSS and Framer Motion.
* When working with Quranic text data, ensure high accuracy and respect for the text format.
* Avoid exposing sensitive API keys or environment variables in the codebase.

## Code of Conduct
Please note that this project is released with a [Contributor Code of Conduct](CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
